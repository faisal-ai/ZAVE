3. IMPORTANT SECURITY NOTES
================================================================

‚ö†Ô∏è  CURRENT IMPLEMENTATION IS CLIENT-SIDE ONLY (FOR DEMO)

‚úÖ FOR PRODUCTION, YOU MUST:

1. CREATE A BACKEND SERVER
   - Node.js, Python, PHP, or any server-side language
   - Never expose secret keys in client-side code
   - Handle payment processing server-side

2. STRIPE BACKEND FLOW:
   a) Client adds items to cart
   b) Client clicks checkout
   c) Client sends order to YOUR server
   d) Your server creates PaymentIntent with Stripe API
   e) Server returns client_secret to client
   f) Client confirms payment with client_secret
   g) Stripe webhooks notify your server of success/failure
   h) Server updates database and sends confirmation email

3. PAYPAL BACKEND FLOW:
   a) Similar to Stripe
   b) Your server creates order
   c) PayPal processes payment
   d) Webhooks notify your server

4. SECURITY BEST PRACTICES:
   - Use HTTPS (SSL certificate)
   - Validate all data server-side
   - Implement CSRF protection
   - Use environment variables for API keys
   - Implement rate limiting
   - Log all transactions
   - Handle webhooks properly

5. HOW TO USE THE CHECKOUT PAGE
================================================================

1. User adds products to cart
2. User clicks "Proceed to Checkout"
3. System checks if user is logged in
   - If not, redirects to login page
   - If yes, redirects to checkout.html
4. User fills shipping information (Step 1)
5. User selects payment method (Step 2)
   - Credit Card (Stripe)
   - PayPal
6. User completes payment
7. System shows confirmation (Step 3)
8. Order saved to localStorage
9. User can track order in account page

9. COMMON ISSUES & SOLUTIONS
================================================================

ISSUE: "Stripe is not defined"
SOLUTION: Make sure <script src="https://js.stripe.com/v3/"></script> 
          is in checkout.html <head> section

ISSUE: "Invalid API Key"
SOLUTION: Check that you copied the full API key correctly
          including the pk_test_ or pk_live_ prefix

ISSUE: "PayPal button doesn't show"
SOLUTION: Check browser console for errors
          Verify Client ID is correct
          Make sure PayPal script loaded

ISSUE: "Checkout page is blank"
SOLUTION: Check that all CSS files are linked:
          - styles.css
          - styles-account.css  
          - styles-checkout.css

ISSUE: "Order doesn't save"
SOLUTION: Check browser console for JavaScript errors
          Make sure user is logged in
          Check localStorage isn't full

11. BACKEND EXAMPLE (Node.js + Express)
================================================================

Here's a simple backend example for Stripe:

// server.js
const express = require('express');
const stripe = require('stripe')('sk_test_YOUR_SECRET_KEY');
const app = express();

app.use(express.json());

app.post('/create-payment-intent', async (req, res) => {
  try {
    const { amount } = req.body;
    
    const paymentIntent = await stripe.paymentIntents.create({
      amount: amount * 100, // Convert to cents
      currency: 'usd',
    });
    
    res.json({ clientSecret: paymentIntent.client_secret });
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

app.listen(3000, () => console.log('Server running'));
